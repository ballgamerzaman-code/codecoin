> อัปเดตล่าสุด: 16 กรกฎาคม 2026 (เว็บ REV 2.1)
> เว็บจริง: **https://ballgamerzaman-code.github.io/codecoin/**
> Repo: **https://github.com/ballgamerzaman-code/codecoin**

## ภาพรวมโปรเจค

CodeCoin คือบอร์ดเกมเพื่อการศึกษา (โปรเจคจบ) ดัดแปลงจาก Monopoly ผสมระบบ QR Code ครอบคลุม 4 วิชา: Coding, Microsoft Office, Graphic Design, การประกอบคอมพิวเตอร์

ผู้เล่นใช้มือถือสแกน QR บนการ์ด/กระดานเพื่อ:
- **ช่องทรัพย์สิน** — อ่านเนื้อหาสั้น + ตอบควิซ ตอบถูกได้ส่วนลด 20%
- **ช่องเหตุการณ์** — สุ่มการ์ดเหตุการณ์ตามโซน (weighted random)
- **ช่องติดบั๊ก (Bug Jam)** — แก้โจทย์ debug จับเวลาเพื่อออกจาก "คุก" ฟรี
- **อัปเกรดทรัพย์สิน** — ตอบคำถามระดับสูงขึ้น (L2 ชำนาญ / L3 ผู้เชี่ยวชาญ) ปลดล็อกค่าเช่าใหม่

ทั้งหมดเป็น static site — ไม่ต้องติดตั้งแอป ไม่ต้อง login โหลดเร็ว host ฟรีบน GitHub Pages

## สถานะ: เนื้อหาเกมครบตามเป้าทุกส่วน ✅

| ส่วน | จำนวน | หมายเหตุ |

| คำถาม | 80 ข้อ | ทุกช่องมี L1×2, L2, L3 + **L4 (ชุดโหมดมั่นใจ ยากกว่า, 25 วิ)** |
| การ์ดเหตุการณ์ | 20 ใบ (โซนละ 5) | mechanic ครบ 4 แบบ: simple / choice / chain / **speed** |

| ผังกระดาน | SVG 28 ช่อง | ธีม PCB REV 2.3 PRINT DARK + **QR จริงสแกนได้ 5 ตำแหน่ง** |

## งานที่ทำเสร็จแล้ว (ตามลำดับเวลา)

### 1. ระบบเว็บหลัก (ต่อยอดจาก handoff)
- แยกข้อมูลเกมออกจากโค้ดเป็น `data.json` — เว็บ fetch ตอนโหลด มี fallback ในตัว (โหลดพลาด → ใช้ข้อมูลสำรอง + แถบเตือน) และซ่อนแถว `DRAFT` อัตโนมัติ
- ระบบอัปเกรด: route `?space={id}&upgrade={level}` เข้าจากปุ่มในหน้าช่องทรัพย์สิน (ใช้ QR ใบเดิม)
- เมนูทดสอบจัดกลุ่มตามโซน มีสีประจำโซนกำกับ
- hook เก็บสถิติรอไว้แล้ว (`CONFIG.log_url` — POST `{kind, space_id, level, correct, ts}`)

### 2. Pipeline ข้อมูลจากทีม content
- `build_data.py` แปลง Excel → data.json: rent 4 คอลัมน์ → array, เฉลย a/b/c/d → index, ตรวจ space_id อ้างอิง, ตรวจเฉลยชี้ตัวเลือกว่าง
- workflow: แก้ `codecoin_database.xlsx` → `py build_data.py` → push (เว็บอัปเดตเองใน ~1 นาที)

### 3. QR และสื่อสิ่งพิมพ์
- `gen_qr.py` สร้าง QR 21 ใบจาก data.json (16 ทรัพย์สิน + event 4 โซน + bugjam)
- การ์ดทรัพย์สิน `codecoin_cards_rev1_2.html`: QR ฝังเป็น data URI (เปิดไฟล์เดี่ยวส่งโรงพิมพ์ได้) ค่าเช่าจริงจากฐานข้อมูล `[เริ่มต้น L0, ชำนาญ L2, ผู้เชี่ยวชาญ L3]`
- ผังกระดาน `codecoin_board_rev2_3_canva.html`

### 4. Deploy
- GitHub Pages จาก branch `main` — push แล้วเว็บอัปเดตเองใน ~1 นาที
- PR #1 (ลิงก์เว็บจริงใน README) merge แล้ว

### 5. ฟีเจอร์เกมบนเว็บ (REV 1.2 → 2.1)
- **ระบบเดิมพันควิซ** (หน้าซื้อทรัพย์สิน): "ตอบธรรมดา" ถูกลด 20% / "มั่นใจ! 🔥" ถูกลด 40% ผิดจ่ายเพิ่ม 10% — ตัวเลขแก้ได้ใน `CONFIG.bet_confident`, log สถิติแยก kind
- **เสียง + visual feedback**: Web Audio ล้วนไม่มีไฟล์เสียง (ติ๊กนับถอยหลังเร่งท้าย 5 วิ, ติ๊งตอบถูก, fanfare โหมดมั่นใจ, โทนต่ำตอบผิด, buzzer หมดเวลา, ฉลองอัปเกรด) + แฟลชเขียว/แดง/ทอง, confetti, ปุ่ม mute ลอย — เสียงปลดล็อกจาก gesture แรกตามนโยบาย browser มือถือ
- **Event แบบ speed** (แข่งตอบเร็วรอบโต๊ะ): โชว์คำถามสุ่มให้ทุกคนแข่งตอบปากเปล่า คนถือมือถือกดปุ่ม "ผู้เล่น 1–4" บันทึกผู้ชนะ → รับรางวัลตาม effect, timer 15 วิ หมดเวลา = ไม่มีใครได้ — ไม่ใช่ multiplayer จริง ไม่มี account/session (เงินจดบนกระดาษนอกเว็บ)
- **Bug Jam เลือกระดับ + hint**: เลือก "ง่าย" (ออกฟรี) หรือ "ยาก 💎" (ถูก = ออกฟรี + โบนัส 300) ก่อนสุ่มโจทย์ / ปุ่ม 💡 ดู hint กดได้ครั้งเดียว — ระดับยากกดแล้วโบนัสเหลือ 150 / ไม่มีโจทย์ hard = ปุ่มยาก disabled พร้อมข้อความ (ตอนนี้รอทีม content เติมแถว hard)
- `build_data.py` รองรับคอลัมน์ใหม่ `difficulty`/`hint` ของชีต bugjam แบบ backward-compatible (ไฟล์เก่าไม่มีคอลัมน์ = easy/ไม่มี hint)
- **สุ่มผลแก้บั๊ก (Fail Roulette, REV 1.6)**: ผูกกับเลนเดิมพัน — "มั่นใจ! + ตอบผิด" บังคับหมุนวงล้อ 7 ช่อง (mult 0.4–1.6 เทียบราคาช่อง) แทนจ่าย flat ธีม debug ไม่ใช่คาสิโน / "ธรรมดา + ผิด" ยังจ่าย flat เดิม / บาลานซ์ EV ~107% (แย่กว่าตอบถูกมั่นใจเสมอ — เดาสุ่มไม่คุ้มกว่าตั้งใจตอบ) ตั้งค่าใน `CONFIG.fail_roulette` reuse `weightedPick`
- **กู้บั๊กด่วน (Debug Rescue, REV 1.7)**: มินิเกม 2 คน 2 มือถือสไตล์ Keep Talking — solver เห็นปริศนา (ตัดสายไฟ), guide เห็นกฎ, ห้ามดูจอกัน คุยด้วยปากเปล่า / **ไม่มี backend**: ทั้งคู่พิมพ์ "รหัสร่วม 4 ตัว" เป็น seed → derive ปริศนา+กฎ+เฉลยชุดเดียวกันแบบ deterministic (seeded PRNG) solver ตรวจผลเองได้ / route `?space=debug_rescue&role={solver|guide}&code=XXXX` / hint เผยกฎ+ตัดเวลา 30% / reward 200 penalty 100 (เบา ตามดีไซน์เน้นสนุกจากการสื่อสาร) / ตอนนี้เล่นได้ pattern `wire_cut` เต็ม อีก 3 pattern (keypad/command_match/final_boss) วางโครง config ไว้ (`enabled:false`)
- **แก้เสียงมือถือติดช้า (REV 1.8)**: `resume()` เป็น async แต่โค้ดเดิมทิ้งโน้ตถ้า context ยังไม่ running → เสียงแรก ๆ หายบนมือถือ — แก้เป็น schedule ต่อ + silent buffer warm-up (iOS) + unlock หลาย event
- **โหมดมั่นใจปรับใหม่ (REV 1.9)**: playtest พบไม่ต่างจากโหมดปกติ — เปลี่ยนเป็น เลือกโหมด**ก่อนเห็นคำถาม** / มั่นใจต้องผ่าน **modal ยืนยัน + countdown 5 วิ** (หมดเวลา/ยกเลิก = กลับเลือกใหม่ ไม่เสียสิทธิ์) / คำถามมั่นใจดึงจาก **L4 ชุดยากแยกต่างหาก** (25 วิ, ไม่ปนกับ logic อัปเกรด) / log เพิ่ม `question_level`
- **Bug Jam หลุดยากขึ้น (REV 2.0)**: playtest พบทุกคนหลุดหมดเพราะลองใหม่ฟรี — ตอบผิด/หมดเวลา = **เสียค่าปรับ** (ง่าย 300 / ยาก 500 + อดโบนัส) ห้ามตอบซ้ำในเทิร์น / **สลับตำแหน่งตัวเลือกทุกครั้ง** (กันจำตำแหน่งเฉลยตอนเล่นซ้ำในห้องเรียน) / เวลาตึงขึ้น: ง่าย 20 วิ ยาก 45 วิ / ปุ่ม "ยาก 💎" เปิดใช้จริงแล้ว (hard 4 ข้อ)
- **Cooling Zone → บอสไฟต์ทั้งวง (REV 2.1)**: จำลอง 12 รอบพบกติกาเดิม (คนหยุดคว้ากองกลางคนเดียว) ตัดสินผู้ชนะด้วยดวง 30% ของเกม — เปลี่ยนเป็น `?space=cooling`: **ทุกคนสแกนเข้าร่วม** เลือก "เลขรอบ" ที่คนหยุดประกาศ → seed ตรงกัน → โจทย์ตัดสายไฟชุดเดียวกันทุกเครื่อง (โหมด solo: เห็นกฎ+สายไฟบนจอตัวเอง, 75 วิ) / ใครผ่านแบ่งกองกลางเท่ากัน — **แผงธนาคาร**ประกาศสัดส่วน (0 คน = สะสมต่อ / 1 = ทั้งกอง / N = คนละ 1/N) เว็บไม่คำนวณเงินจริง / `bossfight_enabled: false` = กลับกติกาดั้งเดิม (ลดอิทธิพลดวง 55% → 32% ตามการจำลอง)

## บั๊กที่เจอและแก้แล้ว

| ปัญหา | วิธีแก้ |
|---|---|
| เนื้อหามีแท็ก HTML ตัวอย่าง (`<h1>`) โดน browser ตีความจนข้อความหาย | เรนเดอร์ฟิลด์เนื้อหาผ่าน `textContent` |
| ตัวเลือกคำถาม `= (เท่ากับ)` โดน Excel เก็บเป็นสูตร ค่าหาย เฉลยเลื่อน | แก้เซลล์เป็นข้อความ + build_data.py ตรวจจับและเตือนเคสนี้ |
| ภาษาไทยพังบน console Windows (cp1252) | บังคับ stdout เป็น UTF-8 ในสคริปต์ |
| ป้ายการ์ด "มือใหม่" โชว์ค่า rent_L0 ขัดกับ database (มือใหม่ = L1) | เปลี่ยนป้ายเป็น "เริ่มต้น" ให้ตรงกระดาน |
| Debug Rescue: กฎบางเวอร์ชันสุ่มได้ layout ที่ทำตามไม่ได้ (v1 มีเหลืองแต่ไม่มีน้ำเงิน, v4 เข้าเงื่อนไขไม่ถึง 2 เส้น) | เพิ่ม `wireRuleClean` วนดึงสีใหม่จนกฎ apply ได้ (rng เดินเหมือนกัน 2 เครื่อง จึงยัง deterministic) fallback เป็น v2 ที่แก้ได้เสมอ |
| มือถือเสียงติดช้า/ไม่ติด (`resume()` async แต่โน้ตโดนทิ้งก่อน context ตื่น) | schedule โน้ตต่อแม้ยัง suspend + silent buffer warm-up + unlock หลาย event |
| ช้อยส์ที่ขึ้นต้น `=` (เช่น `===`, `=SUM`) โดน openpyxl auto-detect เป็นสูตรตอน merge ทำให้คำถามหาย | บังคับ `data_type='s'` ทุกช้อยส์ที่ขึ้นต้น `=` (เช็คจากค่า ไม่ใช่เช็คจาก source type) |
| ไฟล์ทีม content ทำ hint ของ bugjam หายซ้ำ ๆ + `office_02_L1_01` choice_b หายซ้ำ | ทุกรอบรับไฟล์: diff ก่อน merge, re-attach hint จาก repo, restore choice ที่หาย — ห้ามเอาไฟล์ทับตรง ๆ |

> ⚠️ **ฝากทีม content:** ไฟล์ Excel ต้นทางของทีมยังมีบั๊กเซลล์สูตรที่ข้อ `office_02_L1_01` — ให้พิมพ์ตัวเลือกเป็น `'= (เท่ากับ)` (มี apostrophe นำหน้า) ไม่งั้นค่าจะหายทุกครั้งที่ส่งไฟล์ใหม่ ระหว่างนี้เวลารับไฟล์ใหม่ต้อง diff กับของใน repo ก่อนเสมอ อย่าเอาทับตรง ๆ

## งานที่เหลือ

1. **หลัง playtest บอสไฟต์ผ่าน:** เพิ่ม QR `?space=cooling` + เปลี่ยนข้อความบนช่องจุดพักเครื่องในไฟล์ผังกระดาน และ**อัปเดตคู่มือกติกา** (จุดพักเครื่อง: "ใครหยุดคว้ากองกลางคนเดียว" → "ทั้งวงแข่งกู้บั๊ก แบ่งกองกลาง")
2. **ทีม content review hint ของ Bug Jam ระดับ hard 4 ข้อ** (bug_16–19) — ไฟล์ข้อมูลไม่ได้ส่ง hint มา ผมเขียนไว้ให้ก่อน / และแก้ไฟล์ต้นทาง 2 จุดที่พังซ้ำทุกรอบ: คอลัมน์ hint หาย + `office_02_L1_01` choice_b ต้องพิมพ์ `'= (เท่ากับ)` (apostrophe นำหน้า)
3. **Debug Rescue อีก 3 pattern** (keypad / command_match / final_boss) — config วางโครงไว้แล้ว (`enabled:false`) เสียบเพิ่มได้
4. **เก็บสถิติการตอบ** (optional, บทที่ 4 ของเล่ม) — ทำ Apps Script รับ POST แล้วใส่ URL ใน `CONFIG.log_url` — ฝั่งเว็บ log แยก kind ครบ: buy / buy_confident / upgrade / bugjam / bugjam_hard / speed / fail_roulette / debug_rescue / cooling_boss
5. พิมพ์การ์ดจริง: เปิด `codecoin_cards_rev1_2.html` → ปิดเส้น trim สีชมพู → Print เป็น PDF (A4, การ์ด 63×88 มม. + bleed 3 มม.)

## หมายเหตุทางเทคนิค

- เครื่องนี้รัน Python ด้วย `py` (ไม่ใช่ `python`)
- ทดสอบในเครื่องผ่าน `py -m http.server 8000` (fetch ไม่ทำงานบน `file://`)
- Push ต้อง login GitHub ใหม่แต่ละครั้ง (Windows Credential Store มีปัญหา เก็บ credential ถาวรไม่ได้) — ใช้ device flow: กรอกรหัสที่ github.com/login/device
- Repo เป็น public (เงื่อนไข GitHub Pages ฟรี) — เฉลยคำถามเปิดดูได้ถ้ารู้ที่อยู่ repo
- กติกาที่ห้ามเปลี่ยน: naming `{zone}_{ลำดับ 2 หลัก}`, รูปแบบ effect, ธีม PCB, UI ภาษาไทย
