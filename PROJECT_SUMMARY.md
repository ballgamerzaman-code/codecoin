# สรุปงานโปรเจค CodeCoin — QR System

> อัปเดตล่าสุด: 12 กรกฎาคม 2026 (เว็บ REV 1.5)
> เว็บจริง: **https://ballgamerzaman-code.github.io/codecoin/**
> Repo: **https://github.com/ballgamerzaman-code/codecoin**

## ภาพรวมโปรเจค

CodeCoin คือบอร์ดเกมเพื่อการศึกษา (โปรเจคจบ) ดัดแปลงจาก Monopoly ผสมระบบ QR Code ครอบคลุม 4 วิชา: Coding, Microsoft Office, Graphic Design, การประกอบคอมพิวเตอร์

ผู้เล่นใช้มือถือสแกน QR บนการ์ด/กระดานเพื่อ:
- **ช่องทรัพย์สิน** — อ่านเนื้อหาสั้น + ตอบควิซ ตอบถูกได้ส่วนลด 20%
- **ช่องเหตุการณ์** — สุ่มการ์ดเหตุการณ์ตามโซน (weighted random)
- **ช่องติดบั๊ก (Bug Jam)** — แก้โจทย์ debug จับเวลาเพื่อออกจาก "คุก" ฟรี
- **อัปเกรดทรัพย์สิน** — ตอบคำถามระดับสูงขึ้น (L2 ชำนาญ / L3 ผู้เชี่ยวชาญ) ปลดล็อกค่าเช่าใหม่

ทั้งหมดเป็น static site — ไม่ต้องติดตั้งแอป ไม่ต้อง login โหลดเร็ว host ฟรีบน GitHub Pages

## สถานะ: เนื้อหาเกมครบตามเป้าทุกส่วน ✅

| ส่วน | จำนวน | หมายเหตุ |
|---|---|---|
| ช่องทรัพย์สิน | 16 ช่อง (4 โซน × 4) | เนื้อหา + ราคา + ค่าเช่า 4 ระดับ ครบ |
| คำถาม | 64 ข้อ | ทุกช่องมี L1×2, L2×1, L3×1 (ชุด easy v2) |
| การ์ดเหตุการณ์ | 20 ใบ (โซนละ 5) | mechanic ครบ 4 แบบ: simple / choice / chain / **speed** |
| Bug Jam | 15 ข้อ | ทั้ง 4 วิชา + ระดับ easy/hard + hint ทุกข้อ (ตอนนี้ easy ทั้งหมด) |
| QR | 21 ใบ | URL จริง, error correction H, พิมพ์ 3×3 ซม. @300 DPI |
| การ์ดทรัพย์สินพร้อมพิมพ์ | 16 ใบ | QR ฝังในไฟล์ + ค่าเช่าจริง (63×88 มม. + bleed) |
| ผังกระดาน | SVG 28 ช่อง | ธีม PCB REV 2.3 PRINT DARK |

## งานที่ทำเสร็จแล้ว (ตามลำดับเวลา)

### 1. ระบบเว็บหลัก (ต่อยอดจาก handoff)
- แยกข้อมูลเกมออกจากโค้ดเป็น `data.json` — เว็บ fetch ตอนโหลด มี fallback ในตัว (โหลดพลาด → ใช้ข้อมูลสำรอง + แถบเตือน) และซ่อนแถว `DRAFT` อัตโนมัติ
- ระบบอัปเกรด: route `?space={id}&upgrade={level}` เข้าจากปุ่มในหน้าช่องทรัพย์สิน (ใช้ QR ใบเดิม)
- เมนูทดสอบจัดกลุ่มตามโซน มีสีประจำโซนกำกับ
- hook เก็บสถิติรอไว้แล้ว (`CONFIG.log_url` — POST `{kind, space_id, level, correct, ts}`)

### 2. Pipeline ข้อมูลจากทีม content
- `build_data.py` แปลง Excel → data.json: rent 4 คอลัมน์ → array, เฉลย a/b/c/d → index, ตรวจ space_id อ้างอิง, ตรวจเฉลยชี้ตัวเลือกว่าง
- workflow: แก้ `codecoin_database.xlsx` → `py build_data.py` → push (เว็บอัปเดตเองใน ~1 นาที)

### 3. QR และสื่อสิ่งพิมพ์
- `gen_qr.py` สร้าง QR 21 ใบจาก data.json (16 ทรัพย์สิน + event 4 โซน + bugjam)
- การ์ดทรัพย์สิน `codecoin_cards_rev1_2.html`: QR ฝังเป็น data URI (เปิดไฟล์เดี่ยวส่งโรงพิมพ์ได้) ค่าเช่าจริงจากฐานข้อมูล `[เริ่มต้น L0, ชำนาญ L2, ผู้เชี่ยวชาญ L3]`
- ผังกระดาน `codecoin_board_rev2_3_canva.html`

### 4. Deploy
- GitHub Pages จาก branch `main` — push แล้วเว็บอัปเดตเองใน ~1 นาที
- PR #1 (ลิงก์เว็บจริงใน README) merge แล้ว

### 5. ฟีเจอร์เกมบนเว็บ (REV 1.2 → 1.5)
- **ระบบเดิมพันควิซ** (หน้าซื้อทรัพย์สิน): "ตอบธรรมดา" ถูกลด 20% / "มั่นใจ! 🔥" ถูกลด 40% ผิดจ่ายเพิ่ม 10% — ตัวเลขแก้ได้ใน `CONFIG.bet_confident`, log สถิติแยก kind
- **เสียง + visual feedback**: Web Audio ล้วนไม่มีไฟล์เสียง (ติ๊กนับถอยหลังเร่งท้าย 5 วิ, ติ๊งตอบถูก, fanfare โหมดมั่นใจ, โทนต่ำตอบผิด, buzzer หมดเวลา, ฉลองอัปเกรด) + แฟลชเขียว/แดง/ทอง, confetti, ปุ่ม mute ลอย — เสียงปลดล็อกจาก gesture แรกตามนโยบาย browser มือถือ
- **Event แบบ speed** (แข่งตอบเร็วรอบโต๊ะ): โชว์คำถามสุ่มให้ทุกคนแข่งตอบปากเปล่า คนถือมือถือกดปุ่ม "ผู้เล่น 1–4" บันทึกผู้ชนะ → รับรางวัลตาม effect, timer 15 วิ หมดเวลา = ไม่มีใครได้ — ไม่ใช่ multiplayer จริง ไม่มี account/session (เงินจดบนกระดาษนอกเว็บ)
- **Bug Jam เลือกระดับ + hint**: เลือก "ง่าย" (ออกฟรี) หรือ "ยาก 💎" (ถูก = ออกฟรี + โบนัส 300) ก่อนสุ่มโจทย์ / ปุ่ม 💡 ดู hint กดได้ครั้งเดียว — ระดับยากกดแล้วโบนัสเหลือ 150 / ไม่มีโจทย์ hard = ปุ่มยาก disabled พร้อมข้อความ (ตอนนี้รอทีม content เติมแถว hard)
- `build_data.py` รองรับคอลัมน์ใหม่ `difficulty`/`hint` ของชีต bugjam แบบ backward-compatible (ไฟล์เก่าไม่มีคอลัมน์ = easy/ไม่มี hint)

## บั๊กที่เจอและแก้แล้ว

| ปัญหา | วิธีแก้ |
|---|---|
| เนื้อหามีแท็ก HTML ตัวอย่าง (`<h1>`) โดน browser ตีความจนข้อความหาย | เรนเดอร์ฟิลด์เนื้อหาผ่าน `textContent` |
| ตัวเลือกคำถาม `= (เท่ากับ)` โดน Excel เก็บเป็นสูตร ค่าหาย เฉลยเลื่อน | แก้เซลล์เป็นข้อความ + build_data.py ตรวจจับและเตือนเคสนี้ |
| ภาษาไทยพังบน console Windows (cp1252) | บังคับ stdout เป็น UTF-8 ในสคริปต์ |
| ป้ายการ์ด "มือใหม่" โชว์ค่า rent_L0 ขัดกับ database (มือใหม่ = L1) | เปลี่ยนป้ายเป็น "เริ่มต้น" ให้ตรงกระดาน |

> ⚠️ **ฝากทีม content:** ไฟล์ Excel ต้นทางของทีมยังมีบั๊กเซลล์สูตรที่ข้อ `office_02_L1_01` — ให้พิมพ์ตัวเลือกเป็น `'= (เท่ากับ)` (มี apostrophe นำหน้า) ไม่งั้นค่าจะหายทุกครั้งที่ส่งไฟล์ใหม่ ระหว่างนี้เวลารับไฟล์ใหม่ต้อง diff กับของใน repo ก่อนเสมอ อย่าเอาทับตรง ๆ

## งานที่เหลือ

1. **ทีม content เติมโจทย์ Bug Jam ระดับ hard** (คอลัมน์ `difficulty` = hard ในชีต bugjam) — ระบบฝั่งเว็บพร้อมแล้ว ปุ่ม "ยาก" จะเปิดเองเมื่อมีโจทย์
2. **เก็บสถิติการตอบ** (optional, สำหรับบทที่ 4 ของเล่มโปรเจค) — ทำ Apps Script รับ POST ลง Google Sheet แล้วใส่ URL ใน `CONFIG.log_url` ของ index.html — ฝั่งเว็บพร้อมแล้ว (log แยก kind: buy / buy_confident / upgrade / bugjam / bugjam_hard / speed)
3. พิมพ์การ์ดจริง: เปิด `codecoin_cards_rev1_2.html` → ปิดเส้น trim สีชมพูจาก toolbar → Print เป็น PDF (A4, การ์ด 63×88 มม. + bleed 3 มม.)

## หมายเหตุทางเทคนิค

- เครื่องนี้รัน Python ด้วย `py` (ไม่ใช่ `python`)
- ทดสอบในเครื่องผ่าน `py -m http.server 8000` (fetch ไม่ทำงานบน `file://`)
- Push ต้อง login GitHub ใหม่แต่ละครั้ง (Windows Credential Store มีปัญหา เก็บ credential ถาวรไม่ได้) — ใช้ device flow: กรอกรหัสที่ github.com/login/device
- Repo เป็น public (เงื่อนไข GitHub Pages ฟรี) — เฉลยคำถามเปิดดูได้ถ้ารู้ที่อยู่ repo
- กติกาที่ห้ามเปลี่ยน: naming `{zone}_{ลำดับ 2 หลัก}`, รูปแบบ effect, ธีม PCB, UI ภาษาไทย
